# Chilsongmun — 두음법칙 정밀판정 SDK

순수 JavaScript(ESM)만으로 남한어를 북한 문화어로 바꾸는 두음법칙 변환기를 구현했습니다. 조선말규범집 제25항을 기준으로 한자어 어두·어중의 본음을 복원하며, Node.js와 브라우저에서 그대로 동작합니다.

> 이 저장소는 **청류(Cheongryu) 브라우저 확장프로그램**의 「두음법칙 정밀모델」 기능이 사용하는 SDK입니다.
> 엔진(`src/duum.js`, 약 25KB)은 확장프로그램에 내장되고, 대용량 사전 데이터(약 37MB)만
> 이 저장소의 [Releases](../../releases) 데이터팩(`chilsongmun_data_v*.tgz`)에서 최초 1회 내려받아
> 브라우저 안(IndexedDB)에 캐시됩니다. 원격 코드는 일절 내려받지 않습니다(데이터 전용).

## **기능**

* **단어 판정**: `judge("영롱")` 으로 한 단어의 두음법칙 대상 여부와 문화어 표기(`령롱`)를 얻습니다.
* **문장 변환**: `transformText("이력서를 제출하였다")` 로 문장 전체를 어절 단위로 변환하며, 주변 문맥으로 동음이의를 가려냅니다.
* **런타임 겸용**: `import "duum"` 하나로 Node에서는 파일시스템, 브라우저에서는 fetch 로 사전을 로드합니다.

## **동작 원리**

### 두 축 결정론 룩업

표면형만 보고는 판정할 수 없습니다. `이상`은 理想(리상, 변환)일 수도 異常(이상, 유지)일 수도 있어서, 어느 쪽인지는 글자 모양이 아니라 어떤 한자냐에 달려 있습니다. 그래서 **단어→한자**(우리말샘)와 **한자→본음**(Unihan kHangul) 두 개의 결정론 사전을 겹쳐, 단어를 한자로 못 박은 뒤 그 한자의 본음을 복원하는 방식으로 판정합니다.

- 본음이 ㄹ/ㄴ 계열이면 두음법칙이 적용된 것이므로 변환 대상(領=령→영, 女=녀→여)
- 본음이 애초에 ㅇ 등이면 두음과 무관하므로 유지(異=이)

본음↔남한 어두형 변환은 한자별 예외 사전이 아니라 음운 규칙으로 처리했습니다. 음절을 초·중·종성으로 분해해 제10~12항(녀→여, 랴→야, 라→나)을 그대로 적용하므로 어떤 음절에도 일반화됩니다.

### 계층형 단락 평가

판정은 확신이 높은 근거부터 순서대로 시도하고, 확정되면 즉시 반환하는 early-exit 구조입니다. 아래로 갈수록 근거가 약해지므로, 위층에서 걸리면 아래는 아예 보지 않습니다.

- **EXC**: 관용음 예외(나팔·유월·시월 등). 본음이 ㄹ이라도 북한도 두음형으로 적으므로 변환하지 않습니다.
- **NAME**: 인명 성씨 본음 복원(이순신→리순신). 동음이의 성씨가 있어 인명이 확정된 뒤에만 적용합니다.
- **L2**: 우리말샘 한자 + Unihan 본음. 어두·어중을 음절별로 복원하는 핵심 층이며, 동음이의는 아래 WSD 로 가립니다.
- **L율 / L0 / L1 / L2g / L3**: 미등재어를 위한 표면 규칙과 코퍼스·규칙 fallback.

### 동음이의 문맥 해소

한 표면형이 한자에 따라 갈릴 때(`치열`=熾烈/齒列) 빈도만 쓰면 고빈도 한자가 늘 이겨 저빈도 정답을 놓칩니다. 그래서 빈도보다 **문맥 공기어 가중**을 먼저 봅니다.

```
점수 = 문맥가중 × |주변 어휘 ∩ (한자 키워드 ∪ 공기어 사전)| + 예문 수
```

주변에 `경쟁`이 있으면 熾烈로, `치과`가 있으면 齒列로 강하게 기울게 됩니다. 문맥이 겹치지 않으면 자연스럽게 예문 빈도로 떨어집니다.

문맥의 세기는 거리로 조절했습니다. 인접 어절일수록 가중이 크고(dist1=1.0, dist2=0.67, dist3=0.33), 자기 표면형과 같은 이웃은 제외해 반복 표제어가 점수를 오염시키지 않게 했습니다. 또 `와/과/및/쉼표` 같은 등위 경계를 건널 때마다 가중을 감쇠시켜, 옆 절의 단어가 인접 거리로 새어드는 것을 막았습니다.

혼합 동음이의(본음ㅇ과 본음ㄹ/ㄴ이 섞임)에서는 빈도로 밀어붙이지 않습니다. 문맥이 변환 한자를 분명히 지지할 때만 변환하고, 애매하면 보수적으로 비변환을 택하도록 해서 `양자 입양`의 養子(유지)가 고빈도 量子(변환)에 덮이지 않게 했습니다.

### 어중 본음·활음조 복원

두음은 어두에서만 일어나지만, 문화어 표기는 어중도 본음으로 되돌립니다(隊列 대열→대렬, 規律 규율→규률). 그래서 어두 한 음절만 바꾸는 게 아니라 단어를 음절별로 훑으며, 각 한자의 본음 후보 중 두음·활음조를 적용하면 지금 남한 음절이 되는 본음을 골라 복원합니다.

우리말샘에 없는 복합어(`출산율`)는 한자를 못 찾으므로 표면 규칙으로 보완했습니다. 일상 `율`은 한자 본음이 모두 률 계열이라 `율→률` 로 바로 바꿀 수 있지만, `열`은 列/烈(변환)과 熱/說(유지)이 갈려 같은 규칙을 쓸 수 없어 한자 룩업에 맡깁니다.

### 북한어 표제어 실증 병합

우리말샘의 북한어 표제어(조선말대사전 기반 실제 북한 표기)를 남한형 키로 역매핑해 후보에 병합했습니다. 이렇게 하면 남한 표제어에 없는 단어(이사장→리사장)도 변환 근거를 얻고, 선택된 실증 후보는 재계산으로는 못 잡는 관용음(라선형 `용`수철)까지 사전 표기를 그대로 출력하게 됩니다. 다만 실증은 어느 한자를 근거로 삼을지에만 관여하고, 변환/비변환 결정 자체를 뒤집지는 않도록 분리했습니다.

### 런타임 겸용 로더와 데이터

로직 핵심은 코드가 아니라 사전 데이터에 있습니다. 엔진(`duum.js`)은 fetch·fs 의존이 전혀 없는 순수 함수 묶음으로 두고, 사전을 읽어 오는 부분만 런타임별 진입점으로 분리했습니다.

- **Node**: `node.js` 가 `fs` 로 패키지 내장 `data/` 를 읽습니다.
- **브라우저**: `browser.js` 가 `import.meta.url` 기준으로 `fetch` 합니다.
- `package.json` 의 조건부 `exports` 가 `import "duum"` 을 런타임에 맞는 진입점으로 자동 연결합니다.

사전에서 엔진이 실제로 읽는 필드만 남기고 미사용 메타데이터를 덜어내 `word_hanja.json` 을 45MB 에서 34MB(gz 약 8.6MB)로 줄였습니다. 브라우저로 배포할 때는 정적 호스팅의 gzip/brotli 압축을 켜는 것을 권장합니다.

## 사용

### Node.js

```js
import { loadDuum } from "duum";
const j = await loadDuum();
j.judge("영롱").munhwaeo;                       // '령롱'
j.transformText("이력서를 제출하였다").munhwaeo;  // '리력서를 제출하였다'
```

### 브라우저

```html
<script type="module">
  import { loadDuum } from "./output3/src/browser.js";
  const j = await loadDuum();
  console.log(j.transformText("연인").munhwaeo);   // '련인'
</script>
```

`file://` 은 fetch 가 막히므로 정적 서버로 엽니다: `python -m http.server`.

## 예제

```bash
node examples/node-demo.mjs
python -m http.server   # 이후 examples/browser-demo.html 접속
```

## 확장프로그램용 데이터팩 (Releases)

청류 확장프로그램은 아래 릴리스 자산만 내려받습니다.

- 자산 이름: `chilsongmun_data_v<판번호>.tgz` — **라이트판** 사전 JSON 5종을 tar.gz로 묶은 것
- 라이트판은 `node tools/build-datapack.mjs`가 `data/` 전체판에서 만들어냅니다.
  문맥(WSD)이 결과를 바꿀 수 없는 표제어의 공기어(kw)·중복 후보를 제거한 것으로,
  빌더가 전 키(15.4만×2) 전수 대조와 문맥 회귀 배터리로 **판정 동일성을 증명**한 뒤에만 통과시킵니다.
  (v1.0.1 기준: 37MB→10.2MB, tgz 9.1MB→1.7MB, 적재 시간 약 1/3)
- 확장프로그램은 내려받은 파일의 **SHA-256을 내장 값과 대조**한 뒤에만 사용하며, 검증 실패 시 폐기합니다
- 데이터팩 갱신 절차: `data/` 갱신 → `node tools/build-datapack.mjs` (전수 대조 통과 필수) →
  `dist/lite-data/`를 tar.gz(`--format=ustar`)로 묶어 새 태그로 릴리스 →
  확장 쪽 `chilsongmun/offscreen-bridge.js`의 `DATA_VERSION`·`DATA_SHA256` 상수를 함께 올립니다
